# SEO & performance next steps (tolatola.co)

This file is the single checklist for maximizing search visibility and performance for TOLA Tanzania.

---

## What’s already done in code

### SEO
- **Metadata:** Keyword-rich titles and descriptions on all main pages (layout, home, shop, product, shop detail, about, contact, FAQ, vendor/transporter register, terms, privacy, return policy, blog, legal, orders, messages). Canonicals set where relevant.
- **Structured data (JSON-LD):**
  - `WebSite` (name, search action, description)
  - `Organization` (with full NAP: name, address, phone, email, sameAs)
  - `SiteNavigationElement` (sitelinks-style navigation)
  - `FAQPage` on `/faq`
  - `BreadcrumbList` on `/product/[id]` and `/shop/[id]`
- **Technical:** `metadataBase`, `lang="en"`, `theme-color`, preconnect/dns-prefetch for origin, sitemap and robots configured.

### Performance
- **Fonts:** `display: "swap"` on Inter and Playfair (avoids invisible text).
- **Dynamic imports:** `FloatingSupportWidget` and `CookieConsentBanner` loaded client-side only (`ssr: false`) to reduce initial JS.
- **Bundle:** `optimizePackageImports: ["lucide-react"]` in Next.js config for faster dev/build and smaller client bundles.
- **Hero image:** First slide uses `priority` for LCP.

---

## 1. Google Search Console

- [ ] Add and verify the property **https://tolatola.co** in [Google Search Console](https://search.google.com/search-console).
- [ ] Submit **Sitemap:** `https://tolatola.co/sitemap.xml` (after verification).
- [ ] Use **URL Inspection** for key URLs (e.g. `/`, `/shop`, `/about`, `/contact`, `/faq`) and click **Request indexing** so they re-crawl sooner.

Re-crawling and indexing take time; requesting indexing is the main lever to speed it up.

---

## 3. Wikipedia (optional, for Knowledge Panel)

Google often uses Wikipedia for Knowledge Panel content.

- If TOLA or the parent company becomes notable enough, a **Wikipedia article** (written to meet [notability](https://en.wikipedia.org/wiki/Wikipedia:Notability)) can help.
- The article must use the **same NAP** as on tolatola.co and in the Organization schema.
- Do not create a Wikipedia page only for SEO; it must comply with Wikipedia’s policies.

---

## 4. Sitemap and robots

- **Sitemap:** `https://tolatola.co/sitemap.xml` (generated by `app/sitemap.ts`). Submit this URL in Search Console after verifying the property.
- **Robots:** `app/robots.ts` allows crawling and references the sitemap. No extra steps needed unless you change rules.

---

## 5. Performance checklist

- [ ] **Image optimization:** The app currently uses `images.unoptimized: true` (e.g. for standalone/Docker). If you can use Next.js image optimization (no conflicting deployment constraint), set `unoptimized: false` in `next.config.mjs` and use `next/image` with proper `sizes` to improve LCP and bandwidth.
- [ ] **Core Web Vitals:** Monitor in Search Console (Experience → Core Web Vitals) and fix any poor URLs (e.g. LCP, INP, CLS).
- [ ] **Preconnect to API/Supabase:** If your backend and Supabase URLs are fixed in production, add `<link rel="preconnect" href="https://your-api-domain" />` and optionally Supabase origin in `app/layout.tsx` (inside `<head>`). Omit or use env-based logic if domains vary.
- [ ] **Analytics / RUM:** Consider adding a small script or Vercel Analytics / similar to track LCP, FID/INP, CLS in production.

---

## 6. When you change NAP or key SEO

- Update **Organization** in `components/seo/json-ld.tsx`.
- Update **footer** (`components/layout/site-footer.tsx`) and **contact** page so visible NAP matches.
- Re-submit important URLs in Search Console if needed.
- Update this file so the team stays in sync.
